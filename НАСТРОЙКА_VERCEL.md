# Размещение сайта на Vercel (заявки в Telegram работают так же)

Эта инструкция — как перенести сайт с Netlify на **Vercel** с тем же функционалом (заявки приходят в Telegram, опционально архив в Google-таблицу). Для обычного человека, по шагам.

> **Честное предупреждение про Россию.** Vercel, как и Netlify, иногда блокируется в РФ (Роскомнадзор периодически блокирует диапазоны Vercel/Cloudflare). Сейчас Vercel часто открывается без VPN, но 100% гарантии нет. Если и Vercel будет недоступен — самый надёжный вариант для РФ это **российский VPS** (Telegram в России не заблокирован, сервер до него достучится). Скажите — подготовлю и такой вариант.

## Что уже готово в проекте

Я добавил поддержку Vercel прямо в код. Один и тот же сайт теперь работает **и на Vercel, и на Netlify** — менять ничего не нужно:

```
ваша-папка/
├── index.html              ← сайт (форма сама находит нужный адрес функции)
├── api/
│   └── lead.js             ← серверная функция для Vercel  → адрес /api/lead
├── netlify/functions/lead.js  ← то же для Netlify (можно оставить «про запас»)
├── netlify.toml               ← конфиг Netlify (Vercel его игнорирует)
└── .vercelignore              ← говорит Vercel не загружать лишние файлы
```

Форма при отправке сначала пробует адрес Vercel (`/api/lead`), а если его нет — адрес Netlify. Поэтому файлы Netlify можно не удалять.

На Vercel **не нужен** `vercel.json` — он сам понимает: `index.html` это сайт, а всё из папки `api/` — серверные функции.

## Способ 1 — через сайт vercel.com + GitHub (рекомендую, потом обновления в один клик)

### Шаг 1. Аккаунт
1. Откройте **vercel.com** → **Sign Up**. Проще всего войти через **GitHub** (если нет аккаунта GitHub — зарегистрируйтесь на github.com, это бесплатно).

### Шаг 2. Загрузить проект на GitHub
Если не дружите с Git — самый простой путь через сайт GitHub:
1. На **github.com** → кнопка **New** (создать репозиторий) → имя, например `rimma-site` → **Private** → **Create repository**.
2. На странице репозитория → **uploading an existing file** (ссылка в середине) → перетащите **всё содержимое папки** (включая папки `api` и `netlify`) → **Commit changes**.

### Шаг 3. Импорт в Vercel
1. В Vercel → **Add New… → Project**.
2. Выберите ваш репозиторий `rimma-site` → **Import**.
3. Framework Preset оставьте **Other** (ничего не меняйте) → **Deploy**.
4. Дождитесь «Congratulations» — сайт уже в сети по адресу вида `https://rimma-site.vercel.app`.

### Шаг 4. Переменные окружения (см. раздел «Переменные» ниже), затем **Redeploy**.

> Плюс этого способа: чтобы потом обновить сайт, достаточно заменить файл на GitHub — Vercel пересоберёт автоматически.

## Способ 2 — через программу Vercel CLI (без GitHub)

Подходит, если не хотите возиться с GitHub. Нужен установленный Node.js (у вас уже есть).

1. Откройте терминал (PowerShell) и установите CLI:
   ```
   npm i -g vercel
   ```
2. Перейдите в папку сайта:
   ```
   cd "C:\Users\zagva\Desktop\rimma"
   ```
3. Запустите:
   ```
   vercel
   ```
   - Войдите (откроется браузер на vercel.com — подтвердите).
   - На вопросы отвечайте по умолчанию: Set up and deploy? **Y**; Which scope — ваш аккаунт; Link to existing project? **N**; Project name — Enter; In which directory is your code? — **`.`** (точка, Enter); Modify settings? **N**.
4. Добавьте переменные окружения (раздел ниже) и выложите финальную версию:
   ```
   vercel --prod
   ```

## Переменные окружения в Vercel (главное — токен не в коде)

Можно через сайт или через CLI.

### Через сайт
1. Vercel → ваш проект → **Settings** → слева **Environment Variables**.
2. Добавьте по очереди (поле **Key** = имя, **Value** = значение), для **Environment** отметьте все (Production, Preview, Development) и нажмите **Save**:

   | Key | Value | Обязательна |
   |---|---|---|
   | `TELEGRAM_BOT_TOKEN` | токен от @BotFather | да |
   | `TELEGRAM_CHAT_ID` | число от @userinfobot | да |
   | `GSHEET_WEBHOOK_URL` | ссылка Google Apps Script | нет (архив) |

3. **Важно:** после добавления переменных откройте **Deployments → … (три точки у последнего) → Redeploy**, чтобы функция увидела новые значения.

### Через CLI (альтернатива)
```
vercel env add TELEGRAM_BOT_TOKEN
vercel env add TELEGRAM_CHAT_ID
vercel env add GSHEET_WEBHOOK_URL   (если нужен архив)
```
На вопрос про окружение выберите **Production** (можно отметить все). Затем `vercel --prod`.

> Где взять сами значения (токен бота, chat_id, ссылку Google-таблицы) — подробно расписано в файле **`НАСТРОЙКА_ЗАЯВОК.md`**, разделы **A**, **B** и **D**. Значения одинаковые, отличается только место, куда их вставлять (теперь это Vercel, а не Netlify).

## Проверка

1. Откройте свой `https://…vercel.app`.
2. Заполните форму (имя + телефон) → «Отправить заявку».
3. Должно прийти сообщение в Telegram (и строка в таблицу, если настроен архив).

### Если заявка не пришла — куда смотреть
- **Vercel → ваш проект → Logs** (или вкладка **Functions**) — там видно ошибки функции `lead`.
- Частые причины: Римма **не нажала Start** у бота; переменные сохранены с пробелами; после добавления переменных **не сделали Redeploy**.
- Локально (открыт файл `index.html` двойным кликом, адрес `file://`) форма «вживую» не отправит — покажет запасной канал (ВКонтакте). Это нормально: серверная функция есть только на развёрнутом сайте.

## Своё доменное имя (по желанию)
Vercel → проект → **Settings → Domains → Add** → введите домен и пропишите у регистратора записи, которые покажет Vercel. Бесплатный `*.vercel.app` тоже остаётся рабочим.

## Если решите полностью уйти с Netlify
Можно удалить `netlify/` и `netlify.toml` — на Vercel они не используются (а `.vercelignore` и так не даёт их загружать). Но проще оставить: лишним они не мешают, а сайт сможет работать на обеих площадках.
