# Настройка заявок с сайта → Telegram (и архив в Google-таблицу)

Эта инструкция для **обычного человека без технических знаний**. Делайте по шагам, ничего не пропуская. На всё уйдёт 15–20 минут.

## Что в итоге получится

Когда посетитель заполнит форму на сайте и нажмёт «Отправить заявку», вам **мгновенно придёт сообщение в Telegram** с его именем, телефоном и вопросом. По желанию все заявки будут ещё и складываться в **Google-таблицу** (как архив).

Telegram-сообщение будет выглядеть так:

```
🔮 Новая заявка с сайта
━━━━━━━━━━━━━━
👤 Имя: Анна
📞 Контакт: +7-921-123-45-67
✨ Услуга: Натальная карта
🎂 Дата рождения: 15.05.1991, 09:30, Казань
💬 Вопрос: Хочу понять свой жизненный путь.
━━━━━━━━━━━━━━
🕐 04.06.2026 18:30
```

## Как это устроено (коротко, для спокойствия)

Сайт «статический» — у него нет своего сервера. Чтобы безопасно писать в Telegram, нужен секретный **токен бота**. Если вписать его прямо в код страницы — его украдут (код сайта виден всем). Поэтому заявку принимает маленькая **серверная функция на Netlify** (файл `netlify/functions/lead.js`), а токен хранится **только** в защищённых настройках Netlify и в код не попадает. Это бесплатно.

---

## A. Создать Telegram-бота и получить токен

1. Откройте Telegram и в поиске найдите **@BotFather** (с синей галочкой). Нажмите **Start**.
2. Отправьте команду:
   ```
   /newbot
   ```
3. BotFather спросит **имя бота** — напишите любое, например `Заявки Римма`.
4. Затем спросит **username** — он должен заканчиваться на `bot`, например `rimma_zayavki_bot`. Если занят — придумайте другой.
5. В ответ BotFather пришлёт **токен** — длинную строку вида:
   ```
   1234567890:AAEhBP0t...очень_длинная_строка
   ```
   **Скопируйте её и сохраните.** Это пароль от бота — никому не показывайте и не выкладывайте.

---

## B. Узнать chat_id (куда слать заявки)

Бот может писать только тем, кто **первым написал ему**. Поэтому:

1. Откройте в Telegram **своего нового бота** (по username из шага A.4) и нажмите **Start** (или отправьте любое сообщение, например «привет»).
2. Теперь узнаем ваш числовой `chat_id`. В поиске Telegram найдите **@userinfobot**, нажмите **Start** — он сразу покажет ваш **Id** (число вроде `512345678`). Это и есть ваш `chat_id`. Скопируйте его.

**Способ для проверки (необязательно):** откройте в браузере ссылку (вставив свой токен вместо `<ТОКЕН>`):
```
https://api.telegram.org/bot<ТОКЕН>/getUpdates
```
После того как вы написали боту, на странице появится текст с фрагментом `"chat":{"id":512345678,...}` — число после `"id":` это и есть chat_id.

**Если хотите получать заявки в группу:** создайте группу, добавьте туда своего бота, напишите в группе любое сообщение и в `getUpdates` возьмите id группы — он будет **с минусом** в начале (например `-1001234567890`).

## C. Прописать секреты в Netlify (НЕ в код!)

1. Зайдите на **netlify.com**, войдите в аккаунт и откройте ваш сайт.
2. Сверху откройте **Site configuration** → слева в меню **Environment variables**
   (в некоторых версиях: **Site settings → Build & deploy → Environment → Environment variables**).
3. Нажмите **Add a variable** → **Add a single variable** и добавьте по очереди:

   | Key (имя)             | Value (значение)                                  |
   |-----------------------|---------------------------------------------------|
   | `TELEGRAM_BOT_TOKEN`  | токен из шага **A.5**                             |
   | `TELEGRAM_CHAT_ID`    | число из шага **B.2**                             |
   | `GSHEET_WEBHOOK_URL`  | ссылка из раздела **D** (необязательно)           |

4. Нажмите **Save** для каждой.

Эти значения хранятся в защищённой части Netlify и **нигде в публичном коде сайта не видны**.

> Важно: после добавления/изменения переменных нужно, чтобы сайт пере-деплоился (см. раздел E). Если меняете переменные у уже залитого сайта — нажмите **Deploys → Trigger deploy → Deploy site**, чтобы функция увидела новые значения.

## D. (Необязательно) Архив всех заявок в Google-таблицу

Если не нужно — **пропустите этот раздел**, Telegram работает и без него.

1. Создайте новую Google-таблицу. В первой строке сделайте шапку:
   ```
   Дата | Имя | Контакт | Услуга | Дата рождения | Вопрос
   ```
2. В таблице вверху: **Extensions (Расширения) → Apps Script**.
3. Удалите то, что там есть, и вставьте этот код:
   ```javascript
   function doPost(e) {
     try {
       var data = JSON.parse(e.postData.contents);
       var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
       sheet.appendRow([
         data.ts || new Date().toLocaleString('ru-RU', { timeZone: 'Europe/Moscow' }),
         data.name || '', data.contact || '', data.service || '',
         data.birth || '', data.question || ''
       ]);
       return ContentService.createTextOutput(JSON.stringify({ ok: true }))
         .setMimeType(ContentService.MimeType.JSON);
     } catch (err) {
       return ContentService.createTextOutput(JSON.stringify({ ok: false }))
         .setMimeType(ContentService.MimeType.JSON);
     }
   }
   ```
4. Нажмите **Deploy (Развернуть) → New deployment (Новое развёртывание)**.
5. Рядом с «Select type» нажмите шестерёнку → выберите **Web app**.
6. Настройки: **Execute as (Запуск от имени)** = `Me (Я)`; **Who has access (У кого есть доступ)** = `Anyone (Все)`.
7. Нажмите **Deploy**, разрешите доступ (Google попросит подтверждение — это нормально, это ваш же скрипт).
8. Скопируйте полученный **URL веб-приложения** — он вида:
   ```
   https://script.google.com/macros/s/AKfy....../exec
   ```
9. Вставьте этот URL в Netlify как переменную `GSHEET_WEBHOOK_URL` (раздел C).

Если позже захотите отключить архив — просто удалите эту переменную в Netlify. Заявки в Telegram продолжат приходить.

## E. Залить сайт на Netlify и проверить

**Структура папки**, которую загружаете на Netlify (важно сохранить именно так):
```
ваша-папка/
├── index.html
├── netlify.toml
└── netlify/
    └── functions/
        └── lead.js
```

1. Зайдите на netlify.com, перетащите **всю папку** в зону загрузки (Deploys → область «Drag and drop your site folder here»). Либо, если подключён Git — запушьте изменения.
2. Дождитесь окончания деплоя (статус **Published**).
3. **Проверка:** откройте сайт, заполните форму (имя + телефон обязательны) и нажмите «Отправить заявку».
   - На сайте появится «Заявка отправлена ✓» и «Спасибо! Свяжусь с вами в течение 24 часов».
   - В Telegram придёт сообщение. (И строка в таблице, если настроили раздел D.)

### Если заявка не пришла — куда смотреть

- **Netlify → Functions → lead → логи** (Real-time logs). Там видно ошибки.
- Самая частая причина: **Римма не нажала Start** у бота (шаг B.1) — бот не может написать первым.
- Проверьте, что переменные `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID` сохранены **без лишних пробелов**, и что после их добавления сайт пере-деплоился (Deploys → Trigger deploy).
- Проверьте, что токен действующий (не отозван в BotFather).

> На сайте, открытом **локально** (двойным кликом по файлу, адрес `file://`) или не на Netlify, форма работать «вживую» не будет — вместо отправки она вежливо предложит написать во ВКонтакте. Это штатное поведение: серверная функция есть только на Netlify.

## Безопасность (что уже сделано в коде)

- 🔒 Токен бота — **только** в переменных Netlify, в коде сайта его нет.
- 🤖 Защита от ботов (honeypot) — и на странице, и на сервере.
- ⏱️ Ограничение частоты: 1 заявка в 5 секунд (на странице) и 1 в 10 секунд / 5 в час с одного адреса (на сервере).
- 🧹 Текст заявки экранируется перед вставкой в сообщение — нельзя «сломать» бота вредным вводом.
- 📏 Ограничение длины всех полей и на странице, и на сервере.
- 🙈 Данные рождения и контакты идут **только** вам (в ваш Telegram / вашу таблицу) — никаких сторонних сервисов и трекеров.
