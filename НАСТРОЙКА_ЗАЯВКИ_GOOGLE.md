# Заявки → Google-таблица (+ письмо на почту)

Как это работает: посетитель отправляет форму на сайте → данные уходят в **Google Apps Script** →
скрипт **добавляет строку в Google-таблицу** и **присылает письмо** на `rimma.kulmametova@yandex.ru`.
Никакого отдельного сервера (Vercel/Netlify) больше не нужно.

Нужен один раз настроить — минут на 10. Понадобится **аккаунт Google** (Gmail).

---

## Шаг 1. Создать таблицу

1. Откройте https://sheets.google.com и создайте новую пустую таблицу.
2. Назовите её, например, **«Заявки — сайт Риммы»**.
   (Столбцы создавать вручную не нужно — скрипт сделает «шапку» сам при первой заявке.)

## Шаг 2. Вставить скрипт

1. В таблице: меню **Расширения → Apps Script**.
2. Удалите весь текст в открывшемся редакторе.
3. Откройте файл `google-apps-script/Code.gs` из проекта, скопируйте **весь** его текст и вставьте в редактор.
4. Нажмите значок **дискеты** (Сохранить).
5. (По желанию) Если письма на почту не нужны — в строке
   `var NOTIFY_EMAIL = 'rimma.kulmametova@yandex.ru';` замените адрес на пустые кавычки `''`.

## Шаг 3. Развернуть как веб-приложение

1. Вверху справа: **Развернуть → Новое развёртывание**.
2. Нажмите шестерёнку рядом с «Выберите тип» → выберите **Веб-приложение**.
3. Настройки:
   - **Описание:** любое (напр. «Приём заявок»).
   - **Запуск от имени:** **От моего имени** (Me).
   - **У кого есть доступ:** **Все** (Anyone). ← это важно, иначе форма не сможет отправлять.
4. Нажмите **Развернуть**.
5. Google попросит **разрешения** — выберите свой аккаунт → «Дополнительно» → «Перейти к проекту (небезопасно)» → «Разрешить».
   (Это нормально: вы разрешаете доступ собственному скрипту к своей таблице и почте.)
6. Скопируйте **URL веб-приложения** — он заканчивается на `…/exec`.

> Проверка: вставьте этот URL в адресную строку браузера и откройте. Должно показать
> `{"ok":true,"service":"rimma-leads"}`. Значит приложение живо.

## Шаг 4. Вставить URL в сайт

1. Откройте `index.html`, найдите строку (через Ctrl+F: `SHEET_WEBAPP_URL`):
   ```js
   var SHEET_WEBAPP_URL = '';   // '…/exec' from the Apps Script deployment
   ```
2. Вставьте ваш URL в кавычки:
   ```js
   var SHEET_WEBAPP_URL = 'https://script.google.com/macros/s/AKfy.../exec';
   ```
3. Сохраните файл и опубликуйте сайт (загрузите обновлённый `index.html` на хостинг).

## Шаг 5. Проверить

1. Откройте сайт, заполните форму записи тестовыми данными, отправьте.
2. На сайте должно появиться «Спасибо! Свяжусь с вами в течение 24 часов.»
3. В Google-таблице появится новая строка.
4. На почту `rimma.kulmametova@yandex.ru` придёт письмо «🔮 Новая заявка с сайта — …».
   (Первое письмо может попасть в «Спам» — отметьте «Не спам», дальше будут приходить во «Входящие».)

## Частые вопросы

- **Заявка не приходит / на сайте «Не удалось отправить».**
  Чаще всего: в шаге 3 доступ выставлен не «Все», либо в `SHEET_WEBAPP_URL` вставлен не тот URL
  (нужен именно `…/exec`, а не ссылка на редактор скрипта). Проверьте URL по «Проверке» из шага 3.

- **Изменили код скрипта — что делать?**
  Нужно **переразвернуть**: Развернуть → Управление развёртываниями → карандаш (изменить) →
  Версия: «Новая версия» → Развернуть. URL при этом не меняется.

- **Хочу видеть заявки и в Telegram тоже.**
  Можно добавить в скрипт ещё и отправку в Telegram — скажите, допишу.

- **Старые файлы.**
  `api/lead.js`, папка `netlify/`, `netlify.toml`, `НАСТРОЙКА_VERCEL.md`, `НАСТРОЙКА_ЗАЯВОК.md`
  относятся к прежнему варианту через Telegram и больше не используются. Их можно удалить —
  скажите, уберу, чтобы не путались.
