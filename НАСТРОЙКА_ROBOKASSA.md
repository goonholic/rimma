# Подключение оплаты Robokassa

Как работает: на сайте кнопка **«Записаться и оплатить»** → сначала заявка уходит в Telegram
(контакт клиента сохраняется), затем человека перебрасывает на оплату Robokassa → после оплаты
Robokassa подтверждает её на нашем сервере, и в беседу прилетает **«✅ Оплата получена»**.

Код уже готов (`api/pay.js`, `api/robokassa-result.js`, `success.html`, `fail.html`).
Тебе остаётся: заполнить настройки в ЛК Robokassa и вписать ключи в Vercel.

Замени `ТВОЙ-САЙТ` ниже на реальный адрес сайта (например `rimma-astro.ru` или `xxx.vercel.app`).

---

## Часть 1. Личный кабинет Robokassa (у тебя есть доступ)

### 1.1 Технические настройки магазина
В ЛК Robokassa → магазин → **Технические настройки**:

| Поле | Значение |
|---|---|
| **Result URL** | `https://ТВОЙ-САЙТ/api/robokassa-result` |
| Метод Result URL | **POST** |
| **Success URL** | `https://ТВОЙ-САЙТ/success.html` |
| Метод Success URL | **GET** |
| **Fail URL** | `https://ТВОЙ-САЙТ/fail.html` |
| Метод Fail URL | **GET** |
| **Хеш-алгоритм** | **MD5** (важно — код считает подпись по MD5) |

Сохрани.

### 1.2 Пароли
В разделе с паролями возьми и сохрани (понадобятся для Vercel):
- **Пароль #1** и **Пароль #2** (боевые),
- **Тестовый Пароль #1** и **Тестовый Пароль #2**,
- **Идентификатор магазина** (MerchantLogin).

### 1.3 Фискализация (для самозанятой — уточни!)
Robokassa почти всегда требует фискализацию (чек 54-ФЗ). Для самозанятого схема особая
(чек через «Мой налог» либо через Robokassa). **Спроси поддержку Robokassa**, нужна ли передача
блока `Receipt`. Если да — скажи мне, добавлю формирование чека в `api/pay.js` (сейчас он не отправляется).

## Часть 2. Ключи в Vercel

Vercel → проект → **Settings → Environment Variables**. Добавь (Environment: отметь Production):

| Key | Value |
|---|---|
| `ROBOKASSA_LOGIN` | идентификатор магазина |
| `ROBOKASSA_PASS1` | боевой Пароль #1 |
| `ROBOKASSA_PASS2` | боевой Пароль #2 |
| `ROBOKASSA_TEST_PASS1` | тестовый Пароль #1 |
| `ROBOKASSA_TEST_PASS2` | тестовый Пароль #2 |
| `ROBOKASSA_IS_TEST` | `1` — пока тестируем (потом поставить `0`) |

`TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID` уже стоят (от бота) — они же используются для отстука об оплате.

После добавления — **Deployments → ⋯ → Redeploy** (иначе функции не увидят ключи).

---

## Часть 3. Проверка (тестовый режим)

1. Убедись, что `ROBOKASSA_IS_TEST = 1`, сделан Redeploy.
2. На сайте выбери услугу, заполни форму, нажми **«Записаться и оплатить»**.
3. Должно: прийти заявка в Telegram → перебросить на тестовую страницу оплаты Robokassa.
4. Оплати **тестовой картой** (реквизиты — в ЛК Robokassa, раздел тестирования).
5. После оплаты: откроется `success.html`, а в беседу прилетит **«✅ Оплата получена»**.
6. Если что-то не так — смотри Vercel → проект → **Logs** (функции `pay` и `robokassa-result`).

## Часть 4. Включить боевой режим

1. Проверь, что `ROBOKASSA_PASS1` / `ROBOKASSA_PASS2` (боевые) вписаны.
2. Поставь `ROBOKASSA_IS_TEST = 0` (или удали переменную) → **Redeploy**.
3. Сделай одну реальную оплату на минимальную услугу для финальной проверки.

---

## Примечания

- **Цены** заданы на сервере в `api/pay.js` (объект `SERVICES`). Менять цену — только там,
  из браузера сумму подменить нельзя.
- **Безопасность:** Пароли #1/#2 лежат только в переменных Vercel, в коде сайта их нет.
  Оплата считается подтверждённой только если подпись от Robokassa совпала (Пароль #2).
- **Контакт клиента не теряется:** заявка с именем и телефоном уходит в Telegram ещё *до* оплаты,
  поэтому даже при неудачной оплате Римма сможет связаться с человеком.
